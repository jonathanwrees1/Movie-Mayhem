Movie Time Mayhem


This web application will provide users with access to information about different movies, directors, and genres.
Users will be able to sign up, update their personal information, and create a list of their favorite movies.

  • [3] The URL endpoint is /movies/genre/(genre name)
  • This Request searches for a specific genre (by genre name), and returns info about that genre. If the genre does not exist in the database, a (400) error will be thrown with the message,"Could not find that one!"
  • The Method type used is GET.
  • The query parameters used are /movies/genre/:Name
  • There is no data requested. URL Example: localhost:8080/movies/genre/Action
  • The response data is a JSON object with information about that genre.
    Example: { "Name": "Action", "Description": "The genre tends to feature a mostly resourceful hero struggling against incredible odds, which include life-threatening situations, a dangerous villain, or a pursuit which usually concludes in victory for the hero." }



  • [4] The URL endpoint is /movies/director/(director name)
  • This Request searches for a specific director (by director name), and returns information about that director. If the director does not exist in the database, a (400) error will be thrown with the message,"Could not find that one!"
  • The Method type used is GET.
  • The query parameters used are /movies/director/:Name
  • There is no data requested. URL Example: localhost:8080/movies/director/Ted%20Berman
  • The response data is a JSON object with information about that director.
    Example: { "Name": "Ted Berman", "Bio": "American film director, animator, and screenwriter, known for his work with Disney, including Bambi, Fantasia, and The Black Cauldron.", "BirthYear": "1919", "DeathYear": "2001" }



  • [5] The URL endpoint is /users
  • This Request allows a new user to register. Once they fill out the requested info, they will be assigned an ID number, and be added to the API. If the user entered already exists, a (400) error will be thrown with the message,"(UserName) already exists"
  • The Method type used is POST.
  • The query parameters used are /users
  • A body of data is requested which must at least include the user's UserName, Password, and Email in a JSON object. Birthday and Favorite Movies are optional for registration.
    Example: { "UserName": "testusername1" "Password": "testpassword1" "Email": "testemail@gmail.com" "Birthday": "2001-12-13"}
  • The response data is a JSON object, which returns the users newly created profile with their ID number.
    Example: { "UserName": "testusername1" "Password": "testpassword1" "Email": "testemail@gmail.com" "Birthday": "2001-12-1300:00:00.000z" "FavoriteMovies": [] "_id": "6216e1d5d5229e65c7944bc7", "_ _v":0 }



  • [6] The URL endpoint is /users/(UserName)
  • This Request allows users to update their user info by their UserName.
  • The Method type used is PUT.
  • The query parameters used are /users/:UserName
  • The data requested is a JSON object which must contain whatever changes the user wishes to make.
    Example:{"UserName" : "testusername1" "Password": "updatedPassword" "Email": "testemail@gmail.com" "Birthday": "2001-12-13" }
  • The response data is a JSON object, which returns the users updated profile.
    Example:{ _id": "6216e1d5d52a9e65c7944bc7" "UserName": "testusername1" "Password": "updatedPassword" "Email": "testemail@gmail.com' "Birthday": "2001-12-13T00:00:00.000z" "FavoriteMovies":[], "_ _v":0 }



  • [7] The URL endpoint is /users/(UserName)/movie/(Movie ID number)
  • This request allows users to add their favorite movies to their array according to MovieID.
  • The Method type used is POST.
  • The query parameters used are /users/:UserName/movies/:MovieID
  • There is no body of data requested. You simply type in the MovieID you wish to add at the end of the URL. URL Example: localhost:8080/users/testusername1/movies/62153e25828d293b8524adde
  • The response data is a JSON object, which returns your user profile with the new movie added to the FavoriteMovies array.
    Example:{_id": "6216e1d5d52a9e65c7944bc7", "UserName" : "testusername1", "Password": "updatedPassword", "Email": "testemail@gmail.com", "Birthday":"2001-12-13T00:00:00.000Z", "FavoriteMovies":[ "62153e25828d293b8524adde"], "_ _v":0}



  • [8] The URL endpoint is /users/(UserName)/movie/(Movie ID number)
  • This request allows users to remove movies from their array according to MovieID.
  • The Method type used is DELETE.
  • The query parameters used are /users/:UserName/movies/:MovieID
  • There is no body of data requested. You simply type in the MovieID number you wish to delete at the end of the URL. URL Example: localhost:8080/users/testusername1/movies/62153e25828d293b8524adde
  • The response data is a JSON object, which returns your user profile with the selected movie deleted from the FavoriteMovies array.
    Example:{_id": "6216e1d5d52a9e65c7944bc7", "UserName" : "testusername1", "Password": "updatedPassword", "Email": "testemail@gmail.com", "Birthday":"2001-12-13T00:00:00.000Z", "FavoriteMovies":[], "_ _v":0}



  • [9] The URL endpoint is /users/(UserName)
  • This Request allows a user to delete all their information from the API. If a nonexistent UserName is entered, a (400) error will be throw with the message, "(UserName) was not found"
  • The Method type used is DELETE.
  • The query parameters used are /users/:UserName
  • There is no body of data requested. Simply enter your UserName at the end of the URL. URL Example: localhost:8080/users/testusername1
  • The response data is in Text format, with a message that says, "(UserName) was deleted."
    Example: testusername1 was deleted.
