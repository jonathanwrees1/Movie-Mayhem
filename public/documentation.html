Movie Mayhem


This web application will provide users with access to information about different movies, directors, and genres.
Users will be able to sign up, update their personal information, and create a list of their favorite movies.

  • [5] The URL endpoint is localhost:8080/users
  • This Request allows a new user to register. Once they fill out the requested info, they will be assigned an ID number, and be added to the API. If they don't enter a name, a (400) error will be thrown with message ('users need names').
  • The Method type used is POST.
  • The query parameters used are /users
  • A body of data is requested which must at least include the user's name in JSON format.
    Example: { "name": "Mary", "favoriteMovies": "['Episode 1: The Phantom Menace']", "dateOfBirth": "November 5, 1988", "email": "quitecontrary33@gmail.com" }
  • The response data is in JSON format. Which returns the users newly created profile with their ID number.
    Example: { "name": "Mary", "favoriteMovies": "['Episode 1: The Phantom Menace']", "dateOfBirth": "November 5, 1988", "email": "quitecontrary33@gmail.com", "id": "415a0e20-54d0-45b4-9ff7-9ce085da69bd" }



  • [6] The URL endpoint is localhost:8080/users/(ID number)
  • This Request allows users to update their user name in the user info. If they enter invalid information, a (400) error will be thrown with message ('no such user').
  • The Method type used is PUT.
  • The query parameters used are /users/:id
  • The data requested is in JSON format, and must include the users new userName.
    Example:{ "id": 1, "name": "Kim", "favoriteMovies": [], "dateOfBirth": "January 15, 2001", "userName": "kimmy15" (new userName entered), "email": "kimmyjones@yahoo.com" }
  • The response data is in JSON format, which returns the users updated userName while retaining all other info.
    Example:{ "id": 1, "name": "Kim", "favoriteMovies": [], "dateOfBirth": "January 15, 2001", "userName": "kimmy15"(new userName returned), "email": "kimmyjones@yahoo.com" }



  • [7] The URL endpoint is localhost:8080/users/(ID number)/(movie title)
  • This request allows users to add their favorite movies to their array. If they enter invalid information, a (400) error will be thrown with message ('no such user').
  • The Method type used is POST.
  • The query parameters used are /users/:id/:movieTitle
  • There is no body of data requested. You simply type in the movie title you wish to add in the URL after your ID number. URL Example: localhost:8080/users/1/Episode%204:%20A%20New%20Hope
  • The response data is in text format, with a message that says,"(movie title) as been added to user (ID)'s array".
    Example: Episode 4: A New Hope has been added to user 1's array



  • [8] The URL endpoint is localhost:8080/users/(ID number)/(movie title)
  • This Request allows users to remove a movie from their array. If they enter invalid information, a (400) error will be thrown with message ('no such user').
  • The Method type used is DELETE.
  • The query parameters used are /users/:id/:movieTitle
  • There is no body of data requested. You simply type in the movie title you wish to delete in the URL after your ID number. URL Example: localhost:8080/users/1/Episode%204:%20A%20New%20Hope
  • The response data is in text format, with a message that says,"(movie title) has been removed from user (ID)'s array".
    Example: Episode 4: A New Hope has been removed from user 1's array



  • [9] The URL endpoint is localhost:8080/users/(ID number)
  • This Request allows a user to delete all their information from the API. If they enter invalid information, a (400) error will be thrown with message ('no such user').
  • The Method type used is DELETE.
  • The query parameters used are /users/:id
  • There is no body of data requested. Simply enter your ID number at the end of the URL. URL Example: localhost:8080/users/2
  • The response data is in Text format, with a message that says, "user (ID number) has been deleted".
    Example: user 2 has been deleted
